Promiwag

Build & Install

With GODI

Standard GODI Bootstrap

Get GODI's RocketBoost from the official site, unpack it somewhere, and go to that directory. Just pick an empty directory like $HOME/usr/godi312. Promiwag itself should work with OCaml 3.11, or 3.10 too (not tested recently) but these old GODI sections do not provide the right versions of the proving tools (e.g. we need Why >= 2.27). 
./bootstrap --prefix $HOME/usr/godi312 --section 3.12
Add $HOME/usr/godi312/bin and $HOME/usr/godi312/sbin to the $PATH. Then do the stage 2: 
./bootstrap_stage2

Install Promiwag's Dependencies

To compile the Promiwag library, you need: 
godi_console perform \
    -build godi-extlib \
    -build godi-easy-format \
    -build godi-omake
The minimal stuff to prove the theorems on Promiwag's output is: 
godi_console perform \
    -option godi-ocamlgraph:GODI_OCAMLGRAPH_VIEkkWER=no \
    -build apps-alt-ergo \
    -option apps-why:CONF_WHY_USE_COQ=no \
    -option apps-why:CONF_WHY_ENABLE_GUI=no \
    -option apps-why:CONF_WHY_FRAMAC_PLUGIN=no \
    -option apps-why:CONF_WHY_USE_ZENON=no \
    -build apps-why
But if we don't care about the compilation time: 
godi_console perform \
    -option apps-coq:CONF_COQ_DOCUMENTATION=no \
    -option apps-why:CONF_WHY_FRAMAC_PLUGIN=no \
    -build apps-alt-ergo \
    -build apps-why
Achtung! This will track quite a lot of dependencies — e.g. the LablGTK stuff, and the whole Coq framework which takes long to compile, on Ubuntu, I had to apt-get at least: 
apt-get install libgtkgl2.0-dev glutg3-dev tk8.4-dev libxmu-dev

Get MLpcap for Testing

To test the OCaml output, we need MLpcap which is not in GODI. I got mlpcap Ubuntu's package, and: 
dpkg -X libmlpcap-ocaml-dev_0.9-14build1_amd64.deb libmlpcap
cd ./libmlpcap/usr/lib/ocaml/pcap
ocamlfind install pcap *
ocamlfind ocamlopt -package pcap -linkpkg pcap.cmxa pcap_loop.ml
(don't know why pcap.cmxa is needed).
