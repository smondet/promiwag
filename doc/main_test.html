Usage of promiwag_test

Usage:
promiwag_test [-help | OPTIONS] test1 test2
Command Line Options:
  • -iterations <number>
    Number of iterations for benchmarks (default: 1).
  • -pcap-dir <path>
    Directory contain PCap files to process (default: ./datatmp/pcap/).
  • -test-name <name>
    Name to give to the test (default: Test).
  • -handling-style <name>
    Specify user request: full, muted, or light (default: full).
  • -create-access-checks-for-pointers
    Create buffer accessing checks even when only a pointer is requested.
  • -list-tests
    List available tests to call.
Available Tests:
  • C-AST: Some old stuff about generating C code
  • CPS: Generate a Clean Protocol Stack
  • C-bench: Do a benchmark of generated C code
  • MinPars: Generate some minimal parsing code (prints Stiel code)
  • PI: Print the Internet ...
  • why: Some old why output
  • Prove: Launch the proving on a few `borderline' tests
